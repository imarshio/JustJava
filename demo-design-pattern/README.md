# 设计模式

在实际的编程中，我们会接触到很多设计模式，合理的使用设计模式会极大的提高代码的可维护性和可扩展性，且能够更好的服务程序。

## 思考

为什么需要设计模式？之前接触设计模式的时候，感觉只是为了了解而了解，不去思考为什么，所以在一段时间后，了解的其皮面知识就会被遗忘。

现在，我准备换一种方式，重温设计模式，并开始思考，为什么需要设计模式？为什么需要单例模式？为什么需要使用代理模式？
